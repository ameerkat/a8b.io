[Note To Self] Static Site Hosting with S3 and Google Domains

May 15, 2018

Note to self are set up instructions I write mainly for myself.
Most web hosting companies offer web hosting at fixed or tiered costs based on usage, however the lowest tier is usually still at least a few dollars. With Amazon S3 you can get a static website up and running for pennies if your traffic is low. This is great as a low cost way to experiment. Below are the steps on how to get your site up and running and set the proper DNS entries on domains.google (although you should be able to get this working with any registrar).
  1. Find a domain on your registrar of choice, I like to use domains.google so I'll show you how to set DNS entries on there.
  2. If you don't have AWS, sign up and log in. Head to the S3 section.
  3. Create a new bucket that matches your domain using the "www" subdomain. For example for this site I would use "www.a8b.io" as the bucket name.
  4. After you create the bucket, on the properties tab of the bucket enable "Static website hosting." Set the name of the index and error html pages if you want. I usually use the defaults.
  5. Make sure everything you upload that you want to be accessible to the bucket is public. You can upload a simple html test page for now just to ensure the bucket is working.
  6. On google.domains, under the DNS settings for your site, under Synthetic records set up a Subdomain forward for "@.a8b.io" to www.a8b.io
  7. Under custom records, setup a CNAME entry for "www" pointing to the URL of your bucket <bucket name>.s3-website-<region>.amazonaws.com
